## Reflection Report

In this practical, I tried to build a simple TikTok clone using Next.js. The purpose of this task was to understand how a modern web application works and how different pages are connected in one project. In the project, I created several pages such as login, signup, explore, profile and upload page.

I also used some reusable components like VideoFeed and VideoCard. These components help to show the video contents in the application. Using components makes the code easier to manage and reuse in other parts of the project. Next.js was used to handle the routing between pages and CSS was used to design the user interface.

# What I Learned

From this practical, I learned how to create a Next.js project and run it on my computer. I also understood how the folder structure works in Next.js and how each file can represent a page in the application.

Another thing I learned is how to organise the project using folders and components. This makes the project more clear and easier to understand. I also learned how to push my work to GitHub and keep my repository updated.

# Challenges Faced

While doing this practical, I faced some problems. At first, I had difficulty running the development server because some dependencies were not installed. I also got errors when some icons were not imported properly.

Another challenge was understanding the folder structure of Next.js because there were many folders and files which was confusing in the beginning.

# How I Solved Them

To solve these problems, I carefully checked the error messages shown in the terminal. After that, I installed the missing packages and corrected the import statements.

I also checked some online resources and documentation about Next.js to understand the correct structure of the project. After making these changes, the application was able to run properly.

# Conclusion

Overall, this practical helped me understand how a web application can be built using Next.js. It also helped me learn how to organise code using components and folders. I also improved my understanding of debugging errors and using GitHub to manage my project.
